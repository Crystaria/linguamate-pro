## 📋 部署步骤

### 方式 1：手动部署（推荐，15 分钟）

#### 步骤 1：部署前端到 Vercel

**1. 访问 Vercel**
- 打开 https://vercel.com
- 用 GitHub 账号登录

**2. 导入项目**
- 点击 "Add New Project"
- 选择 `linguamate-pro` 仓库
- Root Directory 填：`frontend`
- 点击 "Deploy"

**3. 配置环境变量**
```
REACT_APP_API_URL=https://你的后端.railway.app
```

**4. 等待部署完成**
- 约 2-3 分钟
- 获得域名：`https://linguamate-pro-xxx.vercel.app`

---

#### 步骤 2：部署后端到 Railway

**1. 访问 Railway**
- 打开 https://railway.app
- 用 GitHub 账号登录

**2. 创建新项目**
- 点击 "New Project"
- 选择 "Deploy from GitHub repo"
- 选择 `linguamate-pro` 仓库

**3. 配置服务**
- Root Directory 填：`backend`
- Start Command：`python main_local.py`

**4. 配置环境变量**
```
PORT=8000
HOST=0.0.0.0
DEBUG=False
AI_MODE=local
```

**5. 等待部署完成**
- 约 3-5 分钟
- 获得域名：`https://你的后端.railway.app`

---

#### 步骤 3：更新前端 API 地址

**1. 修改前端配置**

编辑 `frontend/src/config.js`（如不存在则创建）：

```javascript
// 生产环境 API 地址
export const API_URL = process.env.REACT_APP_API_URL || 'http://localhost:8000';
```

**2. 在 Vercel 重新部署**
- 进入 Vercel 项目页面
- Settings → Environment Variables
- 添加 `REACT_APP_API_URL` = 你的 Railway 地址
- 点击 "Redeploy"

### 方式 2：一键部署脚本（高级）

```bash
# 安装 Vercel CLI
npm install -g vercel

# 安装 Railway CLI
npm install -g @railway/cli

# 登录
vercel login
railway login

# 部署前端
cd frontend
vercel --prod

# 部署后端
cd ../backend
railway up
```

## ⚠️ 注意事项

### 1. CORS 配置

确保后端允许前端域名访问：

```python
# backend/main.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:3000",
        "https://你的前端.vercel.app"  # 添加生产域名
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 2. 文件大小限制

Vercel 免费限制：
- 请求体最大：6MB
- 函数执行时间：10 秒

Railway 免费限制：
- 每月 $5 额度
- 内存：512MB
- CPU：共享

### 3. 数据库

当前版本使用内存存储，重启后数据丢失。

**建议：** 添加 SQLite 或 Railway PostgreSQL

## 🆘 常见问题

### Q: Vercel 部署失败？

A: 检查：
- Node.js 版本是否 16+
- 是否正确设置 Root Directory 为 `frontend`
- `package.json` 是否有 build 脚本

### Q: Railway 部署失败？

A: 检查：
- `requirements.txt` 是否完整
- Start Command 是否正确
- 端口是否设置为 8000

### Q: 前端无法连接后端？

A: 检查：
- CORS 配置是否包含前端域名
- API URL 是否正确
- 后端服务是否正常运行

### Q: 图片上传失败？

A: 检查：
- 文件大小是否超过限制
- Railway 是否有足够额度
- 考虑使用云存储（如 Cloudinary）
